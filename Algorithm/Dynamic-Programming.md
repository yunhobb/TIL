# 다이나믹 프로그래밍


```한번 계산한 문제는 다시 계산하지 않도록 하는 알고리즘```
- 메모리 공간을 약간 더 사용하여 연산속도를 비약적으로 증가시킬 수 있다.

## 사용조건
1. 최적 부분 구조: 큰 문제를 작은 문제로 나눌 수 있다.
2. 중복되는 부분 문제: 동일한 작은 문제를 반복적으로 해결한다.

## 1.메모이제이션(Memoization)
  - 한 번 게산한 결과를 메모리 공간에 메모하는 기법
    - 같은 정보를 다시 호출하면 메모했던결과를 그대로 가져옴
    - 값을 기억해둔다는 점에서 캐싱(cahing)라고 함

## 2.탑다운 vs 보텀업
   - 큰 문제부터 해결해서 작은 문제를 해결해 나감(탑다운)
   - 작은 문제부터 해결해서 큰 문제를 해결(보텀업)
   
### DP 정리
> * 다이나믹 프로그래밍의 전형적인 형태는 보텀업 방식
> * 보텀업 방식에서 사용되는 결과 저장용 리스트는 'DP 테이블'이라 부릅니다.
> * 메모이제이션은 탑다운 방식에 국한 되어 사용되는 표현

## 문제를 대하는 방법
1. 주어진 문제가 다이나믹 프로그래밍 유형인지 파악
2. 특정한 문제를 완전탐색으로 접근했을 때 시간이 매우 오래 걸리면 다이나믹 프로그래밍을 적용할 수 있는지 확인합니다.
    - 중복되는 부분문제들이 있는지 확인하는게 중요
3. 일단 단순히 재귀 함수로 비효율적인 프로그램을 작성한 뒤 (탑다운) 작은 문제에서 구한 답이 큰 문제에서 그대로 사용될 수 있으면, 즉 메모이제이션을 적용할 수 있으면 코드를 개선하는 방법도 좋은 아이디어
4. 가능하다면 재귀함수를 이용하는 탑다운 방식보다는 보텀업 방식으로 구현을 권장
